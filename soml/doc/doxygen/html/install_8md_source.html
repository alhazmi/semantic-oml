1 OML: The O? Measurement Library: Installation Instructions {#install}
2 ==========================================================
3 
4 This guide documentates building OML from a source tarball or Git clone,
5 as well as the generation of an Android tree suitable for use with
6 CyanogenMod. Please refer to file [README](@ref index) for instructions on using
7 using these programs.
8 
9 
10 For users
11 ---------
12 
13 If you just want to use OML, you probably downloaded this package as a
14 tarball. You'll however need some additional software packages (popt,
15 sqlite3, libxml2, postgresql), and their development headers. They
16 should be available from your distribution, e.g., for Debian,
17 
18  $ sudo apt-get install libxml2-dev libpopt-dev libsqlite3-dev \
19  pkg-config libxml2-utils
20  $ sudo apt-get install libpq-dev # Optional, for PostgreSQL support
21 
22 Then, the standard UNIX `./configure && make` method works for this
23 package. Here's what to do:
24 
25  $ tar xvfz oml2-2.x.x.tar.gz
26  $ cd oml2-2.x.x
27  $ ./configure --help
28 
29 (to see the configuration options), and then:
30 
31  $ ./configure [possible options]
32  $ make
33  $ sudo make install
34 
35 The most useful option to `configure` is the `--prefix` option, which
36 allows installation in a non-standard location. The default is
37 `/usr/local`. If you just use the default, the liboml2 library will be
38 installed in `/usr/local/lib`, and you should be able to link your
39 programs against it with a simple `-loml2` on the command line to the
40 linker. The server, proxy server, and oml2-scaffold will be installed in
41 `/usr/local/bin`, and should be in your standard `PATH`.
42 
43 Another set of useful options are the `--with-pgsql*` ones. They allow
44 to specify the location of the PostgreSQL headers and sources. Most
45 notably, on Debian-based systems,
46 `--with-pgsql-inc=/usr/include/postgresql` is needed.
47 
48 To install OML in your home directory, use:
49 
50  $ ./configure --prefix=${HOME}/usr # For example
51  $ make
52  $ sudo make install
53 
54 Then you will probably need to add ${HOME}/usr/lib to your
55 `LD_LIBRARY_PATH` environment variable. To run the server, add
56 `${HOME}/usr/bin` to your `PATH` environment variable
57 
58 The last command will also install the man pages. Starting points in the
59 documentation are:
60 
61  $ man 1 liboml2
62  $ man 3 liboml2
63  $ man 5 liboml2.conf
64 
65 All functions in the API have their own man pages (see [liboml2(3)](http://oml.mytestbed.net/doc/oml/2.12/liboml2.3.html) for
66 starters).

69 For developers
70 --------------
71 
72 If you want to modify or enhance OML, you probably checked out the
73 sources from the OML git repository at
74 
75  $ git clone git://mytestbed.net/oml.git
76 
77 In that case, in order to build the package you first need to generate
78 the `configure` script and Makefiles. To do this, in addition to the
79 dependencies listed in the first section, you also need the Autotools
80 installed: Autoconf, Automake, Libtool as well as Gnulib. If you use a
81 Linux distribution, these should all be available via your system's
82 package manager. For example on Debian-ish systems:
83 
84  $ sudo apt-get install autoconf automake libtool gnulib
85  $ sudo apt-get install asciidoc texinfo \
86  docbook-xml docbook-xsl xsltproc # Optional, for building documentation
87 
88 Once you have installed the Autotools you should be able to generate all
89 the necessary configuration and Makefiles by using the `autogen.sh` script.
90 
91  $ cd oml
92  $ ./autogen.sh
93 
94 At this point you should be ready to run
95 
96  $ ./configure && make && sudo make install
97 
98 as above.
99 
100 By default the documentation will be built. The documentation is written
101 in Asciidoc markup language, so you'll need Asciidoc installed (`apt-get
102 install asciidoc` on Debian/Ubuntu). Code documentation based on Doxygen
103 markup is also generated. To disable building the documentation, use the
104 `--disable-doc` option to the configure script. To only disable Doxygen
105 documentation, use the `--disable-doxygen-doc` option.
106 
107  $ ./configure --without-doc
108 
109 If you want to run the test suite you will also need to install the
110 CHECK package (a unit testing framework) [check]. It's also available
111 as a Debian package in the standard Debian and Ubuntu repositories:
112 
113  $ apt-get install check
114 
115 If you don't install CHECK then the test suite won't be run, but OML
116 should still build and install correctly. To run the test suite, use the
117 `check` target:
118 
119  $ make check
120 
121 If you prefer to keep your executables, object files, and other
122 generated files in a separate tree, Automake supports a parallel build
123 using GNU Make's `VPATH` feature. For instance, to build in a
124 subdirectory called 'build', do the following:
125 
126  $ mkdir _build
127  $ cd _build
128  $ ../configure [possible options]
129  $ make
130  $ sudo make install
131 
132 All the files generated by `configure` and `make` will be put under the
133 build directory. Automake recognizes that `configure` was run in a
134 directory different to the root of the source tree, and adjusts its
135 behaviour accordingly.
136 
137 Finally, if you want to restore the tree to a clean state, without the
138 generated Makefile, Makefile.in, and copied Autoconf/Automake/Libtool
139 files, do this:
140 
141  $ ./autoclean.sh
142 
143 Once you do this, you'll have to follow the above procedure again to get
144 the tree into a state where you can build it.

147 Android builds
148 --------------
149 
150 Building OML for Android consists in leveraging the native process. This
151 method requires a different set of source files, and cannot work with
152 your current tree. If you are working from a Git checkout, you can
153 generate this set of source files by running the following commands.
154 
155  $ ./configure --enable-packaging
156  $ make pkg-and
157 
158 This will create an archive named p-android/oml2-2.x.x-android.tar.gz.
159 Alternatively, you can find this archive at [oml-files]. That fileset
160 contains both the OML and libpopt source, as well as a [README](@ref index).oml2 file
161 documenting the build process in more details. Please refer to it for
162 further information on that matter.
