Detailed Description

Multi-typed variable container without type information.

WARNING: OmlValueU MUST be omlc_zero()d out before use. Additionally, if the last type of data they contained was an OML_STRING_VALUE, OML_BLOB_VALUE or OML_VECTOR_*_VALUE, they should be omlc_reset_(string|blob|vector)(). Not doing so might result in memory problems (double free or memory leak).

When wrapped in OmlValue, the right thing is done, by the initialisation/reset functions.
