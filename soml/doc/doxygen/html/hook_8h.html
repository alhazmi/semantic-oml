void hook_setup ( void  )

Initialise the event hook if specified.

This function creates two pipes, forks, redirects the child's stdin and stdout to/from these pipes, then executes the hook program

The hook program is expected to first print an identifying banner (

See Also
HOOK_BANNER), then wait for commands on stdin.

Though a reverse pipe is also created for the hook's stdout to be available to the main server process, it is not currently used for anything else than getting the banner. Also, a read(2) on it from the main process is blocking.
