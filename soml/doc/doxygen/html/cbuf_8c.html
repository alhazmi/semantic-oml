Detailed Description

A FIFO implementation used by the proxy that uses a circular list of pages to manage the memory.

It always writes in the tail buffer and always reads from the read buffer; tail->next points back to the start of the buffer:

*  +------------------------------------------+
*  V                                          |
* [X] -> [X] -> ... [X] -> ... -> [X] -> [X] -+
*                    ^                    ^
*                    |_ read              |_ tail
*

When the tail buffer fills up, add a new one after tail, link the new tail to the head, and keep writing.

To remove a buffer from the head, link the tail to the next one.

Could also implement this as a circular chain where beyond a certain size the chain does not expand, by having an empty flag on the nodes and allowing the tail to process, rather than inserting new nodes at the tail.
