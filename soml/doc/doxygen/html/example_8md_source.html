1 Example sine generator reporting over OML {#example}
2 =========================================
3 
4 This simple example demonstrates both a simple OML-instrumented application, as
5 well as the code-generating capabilities of [oml2-scaffold(1)](http://oml.mytestbed.net/doc/oml/2.12/oml2-scaffold.1.html). See
6 [oml-scaffold-tute] for more details.
7 
8 Most of the code has been generated with the following few steps.
9 
10  $ oml2-scaffold --app generator # Create template description file
11  Created 'generator.rb'
12  $ vim generator.rb
13  [edit measurement points and options]
14  ^]:wq
15  $ oml2-scaffold --main generator.rb # Create template C code
16  Created 'generator.c' and 'version.h'
17  $ vim generator.c
18  [add missing includes and sine-generation logic]
19  ^]:wq
20  $ oml2-scaffold --make generator.rb # Create template Makefile
21  $ vim Makefile
22  [add -lm to LIBS, necessary for sin(3)]
23  ^]:wq
24 
25 The binary can be built with a simple call to `make`. In the process,
26 [oml2-scaffold(1)](http://oml.mytestbed.net/doc/oml/2.12/oml2-scaffold.1.html) will be used twice more, as called by the default Makefile
27 template:
28  - `generator_oml.h`, based on the `defMeasurement`s in the description file, which
29  contains the declaration of the measurement points, as well as helper
30  functions to register them to OML and inject samples;
31  - `generator_popt.h`, base on the `defPropertys` in the description file, which
32  contain the basic option-parsing logics for the declared properties (not the
33  `--oml-*` ones, which are directly handled by the OML client library, though
34  options can be given in any order on the command line).
35 
36 Using diff(1) on the distributed files, as compared to those generated by the
37 sequence of commands above, can further show how much of the code can be
38 generated automatically, so the developer can concentrate on the logic of the
39 application, rather than the samples management and reporting tasks.
40 
41 The `config_*.xml` files are examples, usable with the generator, of how OML
42 Measurement Streams can be directed to one or more collection point, and
43 filterred or further processed along the way.
44 
45 To build this example without installing the OML components system wide, run
46  make CC="$PWD/../../libtool compile gcc" CCLD="$PWD/../../libtool link --tag=CC gcc" \
47  SCAFFOLD="ruby $PWD/../../ruby/oml2-scaffold" \
48  CFLAGS="-I$PWD/../../lib/client -I$PWD/../../lib/ocomm" \
49  LDFLAGS="-L$PWD/../../lib/client/.libs -L$PWD/../../lib/ocomm/.libs"
50 
51 The generator can then be run with, e.g.,
52  LD_LIBRARY_PATH=$PWD/../../lib/client/.libs:$PWD/../../lib/ocomm/.libs \
53  ./generator -n 10 -i .5 --oml-id generator --oml-domain test
